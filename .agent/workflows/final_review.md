# Final Delivery Quality Assurance Protocol

ユーザーへの納品（コード、ドキュメント、UI）を行う直前に、以下のチェックリストを**必ず**実行すること。

## 1. 言語チェック (Language Check)
- [ ] タイトル、見出し、本文はすべて**日本語**か？ (Thinking process is English, Output is Japanese)
- [ ] 専門用語は直感的な日本語に翻訳されているか？ (e.g. "Deploy" -> "公開", "Commit" -> "保存")
- [ ] UIのボタン名、エラーメッセージ、ログ出力は日本語化されているか？

## 2. デザイン・品質 (Design & Quality)
- [ ] プレースホルダー（"ここにテキストを入れる"等）は残っていないか？
- [ ] UIデザインは「Premium」で「Glassmorphism」などの指定に従っているか？
- [ ] フォントは `Noto Sans JP` が指定されているか？

## 3. ユーザー視点 (User Centricity)
- [ ] 「どうやって(How)」ではなく「何ができるか/なぜ必要か(What/Why)」が説明されているか？
- [ ] 非エンジニアでも理解できる平易な言葉か？
- [ ] ワンクリックで実行できる形（.batファイル等）で提供されているか？

## 4. 安全性 (Safety)
- [ ] 破壊的な操作（削除・上書き）の前に、バックアップまたは確認フローが存在するか？

---
**判定**: すべての項目が [x] にならない限り、`notify_user` を実行してはならない。
